Temple
 

Temple Library for ActionScript 3 by MediaMonks

ActionScript 3 toolkit for stable, high performance and maintainable Flash applications. Special designed for general everyday development.

This is version 3.3.0 of the Temple, build on 2012-12-13. Get the latest version from: http://code.google.com/p/templelibrary/. You can get the files from SVN or you could download it as a zip from the downloads page.

The Temple consists of classes we use on a regular basis. They are designed for re-usability and optimized for performance and memory usage. The Temple is specially designed to work with other frameworks like Gaia, Robotlegs or TweenLite.

The Temple focuses on:

  • Memory Management
  • Destruction
  • Debugging
  • Utilities
  • UI Components
  • Forms
  • Data loading, parsing and handling
  • Layout
  • Caching

But the Temple contains many other classes which can help you in work as a Flash Developer. The Temple can also be useful for Flash Animators for creating easy, fast and stable interactive UI Components.

The Temple is specially designed to work with Flash in the Flash IDE, but can also be used in combination with Flex.

The Temple is build by MediaMonks and is partly based and inspired on other frameworks and libraries.

Modules

The Temple consists of several modules which all have there own functionality and can be used together. The most common modules are grouped together in the root /source folder of the Temple Library.

Most common modules:

  • Core
  • Common
  • Utils
  • Data
  • Control
  • UI
  • Yalog
  • TweenLite

Other modules:

  • Camera
  • CodeComponents
  • Facebook*

  • Microphone
  • Multi-Language*

  • Sound*
  • Speech

Content

/bin
Contains the SWC files of the Temple.
/doc
Contains the documentation of the Temple.
/examples
Contains examples of the Temple.
/lib
Contains SWC files needed for the Temple.
/source
Contains the source (.as) files of the Temple. These are the source files of the most common modules of the Temple. These files are set as SVN external, so they are directly linked to the original source files in the corresponding module.
/extended
Contains additional source (.as) files of the Temple, from the most common Temple modules. These files are set as SVN external, so they are directly linked to the original source files in the corresponding module.
/modules
Contains all Temple modules. The most common modules are also linked in the source folder of the Temple. The modules folder also contains some more additional modules, like Facebook or CodeComponents.
(This folder is not available in the zip file.)
/tools
Contains ANT scripts which are used to generate docs, examples, swc etc...
(This folder is not available in the zip file.)

Getting started

Adding the Temple to your project

To add the Temple to your project you could use the SWC file or the AS files. The SWC file can be found in the bin folder. If you want to use the SWC file you need to add the SWC file to the class path of your project.

If you want to use the AS files you just need to copy the content of the /source folder to the source folder of your project. Or you could add the /source folder to the class path of your project.
NOTE: don't copy folders from an SVN check out, since this can cause SVN problems. It would be better to do an SVN export to your own folder.

Use the Temple in your project

To get more information about the usage of the classes in the Temple you could check out the AS Documentation, the examples or the readme.html in every module you want to use.

Examples

Check out the /examples folder for all examples. This folder also contains a list of all available examples.

Note: every example has an XML file which is an ANT build file which can be used to build the example.

Version

This is version 3.3.0 of the Temple. This version number consists of 3 numbers which has the following meaning:

  1. The first number incremented when the Temple is completely refactored and will not be backwards compatible with the previous version of the Temple.
  2. The second number is incremented when something has change in the API of the Temple of when classes are added to the Temple. These updates are not intended to be backwards compatible, but should be interchangeable with some minor changes.
  3. The third number is incremented when there were updated in classes which has no effect on the API of the Temple. These updates should be backwards compatible, although backwards compatibility is not supported in the Temple.

When the first or second number is incremented, the following numbers will be reset to zero. For maintainability all modules have the same version number, even when nothing has changed in the module. Modules with the same version number can always be used together.
