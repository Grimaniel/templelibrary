Temple UI Module
 

About

This is version 3.5.1 of the Temple UI Module, built on 2013-06-12. Get the latest version from: http://code.google.com/p/templelibrary/. You can get the files from SVN or you could download it as a zip from the downloads page.

UI (User Interface) is the space where interaction between humans and machines occurs. The UI module of the Temple contains visible elements ("UI Components") which can be added on the stage. This can be elements like buttons, form components, labels, ScrollPanes or Sliders. And also classes for handling animations, forms or layout. Check out the documentation for a list of all classes of the UI module.

Buttons

A button is a clickable interactive user interface element, which is common used for navigation or user input. To make the user aware of the interactivity of the button, the button often reacts on the mouse cursor. This can be done by changing the appearance of the button, like changing color, shape or size.

There are 5 properties which are used to define the 'state' of the button:

  • over the mouse is currently over the button
  • down the user presses with the mouse button on the button
  • selected the button is currently selected
  • disabled the button is currently disabled.
  • focus the button currently has focus.

These states can be combined. A ButtonBehavior is used to manage this 'state'. Whenever this state changes the ButtonBehavior dispatches a ButtonEvent on the button. This event is also dispatched on all children, grant-children etc. of the button. This is called 'event tunneling', which is handled by the EventTunneler.

Other behaviors reacts on these ButtonEvents and can manipulate the button to react in this state. The ButtonTimelineBehavior plays a timeline animation or jump to a specific frame according to the state.

Example of a MultiStateButton which uses ButtonTimelineBehavior

The ButtonTimelineLabels class contains all possible labels which can be used on the timeline with the ButtonTimelineBehavior.

The ButtonStateBehavior can show or hide children on the displaylist of the button to show the state of the button. The ButtonTweenBehavior uses TweenMax to tween the button according to the state. Since these are all behaviors they can easily added to objects. But the Temple has several buttons with these behaviors built-in.

hand
The mouse cursor changes in a hand on mouse-over.
enable
The button can be disabled.
hitarea
A child with the name 'mcHitArea' will automatically be set as HitArea
behavior
The ButtonBehavior keeps track of the status (over, down, selected, disabled) of the button and automatically sends changes to all children, so buttons can be nested.
timeline
ButtonTimelineBehavior, the different statuses of the button can be set by using labels on the timeline.
states
ButtonStateBehavior, children with a certain state will automatically be shown or hidden, dependent on the ButtonStatus.
liquid
LiquidBehavior, automatic positioning and scaling by using properties like 'left', 'right', 'top', 'bottom', 'verticalCenter' etcetera.
label
The button has a text that can be placed.
padding
The button automatically adapts to the size of the button and the text by using LiquidBehavior and padding.
select
The button is selectable.
toggle
Switching between selected/deselected on click is possible.
group
The button can be used in a RadioGroup.
tween
Used TweenMax to switch between different states. TweenVars can be set for every state.
css
The button is added to the StyleManager by default and can be styled with CSS.
url
The button can be directly used for opening a URL by using a name and the URLManager.
form
The button in meant for use in forms.
value
The button has a value (and can be validated).
error
The button can show errors.
focus
The button can show focus and can be tabbed to it in a form.
data
The button has a data property.
component
The button will automatically used in a FormComponent.
submit
In the FormComponent, the button is automatically set to submit the form.
reset
In the FormComponent, the button is automatically set to reset the form.
list
The button is automatically used as ListRow in a list.

Content

This module contains the following folders:

/bin
Contains the SWC of this module.
/doc
Contains the documentation of the UI module.
/examples
Contains some examples of the UI module.
/lib
Contains the SWC files of other Temple modules which are needed for this module.
/source
Contains the source (.as) files of this module.
/extended
Contains the source (.as) files of some additional classes of this module. These additional class have more dependencies to other modules which are not supplied with this module.
/tools
Contains ANT scripts which are used to generate docs, examples, swc etc...
(This folder is not available in the zip file.)

Dependencies

This module is dependent on the core, common, utils and data modules. The SWCs of the these modules can be found in the /lib folder.

Classes inside the /extended folder do have more dependencies to other Temple modules, like the tweenlite module. The SWC files of these modules are not supplied with this module. If you want to use these additional classes you will need these other modules. The modules you need varies per class. Check the imports of the class to define the modules you need.

Getting started

Adding the UI module to your project

To add the UI module to you project you could use the SWC file or the AS files. The SWC file can be found in the /bin folder. If you want to use the SWC file you need to add the SWC file to the class path of your project. You also need to add the dependent SWC's to the class path of your project. These SWCs can be found in the /lib folder.

If you want to use the AS files you just need to copy the content of the /source folder to the source folder of your project. Or you could add the /source folder to the class patch of your project.
NOTE: Don't copy folders from an SVN check out, since this can cause SVN problems. It would be better to do an SVN export to your own folder.

Examples

Check out the /examples folder for all examples. This folder also contains a list of all available examples.
