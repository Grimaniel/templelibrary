Temple CodeComponents Module
 

About

This is version 3.5.1 of the Temple CodeComponents Module, built on 2013-05-23. Get the latest version from: http://code.google.com/p/templelibrary/. You can get the files from SVN or you could download it as a zip from the downloads page.

CodeComponents are 'code-only' UI Components which are full-featured, lightweight and easy to use. The CodeComponents are special design for quick prototype development. The CodeComponents extends the regular Temple components and have the same features as their super classes. Since they extend these Temple classes they can be used in combination with other Temple objects and can (later in the project) easily be replaced for other objects. This makes them very useful for quickly development.

The CodeComponents can be used with Flash and Flex and are not dependent on any special library or compile setting, except for some other Temple module (see Dependencies). All CodeComponents have a 'Code' prefix in their name.

Creation

Create a CodeComponent is as easy as:

// Create a code-only button
this.addChild(new CodeButton(14, 14, 10, 10));

// Create a code-only button with a label
this.add(new CodeLabelButton("TEST"), 10, 10);

// Create a code-only ComboBox
var comboBox:CodeComboBox = new CodeComboBox();
this.addChild(comboBox);
comboBox.x = 20;
comboBox.y = 20;
// add some items to the ComboBox
comboBox.addItems(["Lorem ipsum", "dolor sit amet", "consectetur", "adipiscing", "elit", "Ut rhoncus", "malesuada"]);

// Create a code-only RadioGroup with buttons
var group:RadioGroup = new RadioGroup();
group.add(CodeRadioButton(this.add(new CodeRadioButton("This is option 1", 1), 10, 10)));
group.add(CodeRadioButton(this.add(new CodeRadioButton("This is option 2", 2), 10, 30)));
group.add(CodeRadioButton(this.add(new CodeRadioButton("This is option 3", 3), 10, 50)));
group.add(CodeRadioButton(this.add(new CodeRadioButton("This is option 4", 4), 10, 70)));

Skinning

The CodeComponents can be skinned by changing the properties of the CodeStyle class. This must be done before you create the components.

Content

This module contains the following folders:

/bin
Contains the SWC of this module.
/doc
Contains the documentation of the CodeComponents module.
/examples
Contains some examples of the CodeComponents module.
/lib
Contains the SWC files of other Temple modules which are needed for this module.
/source
Contains the source (.as) files of this module.
/tools
Contains ANT scripts which are used to generate docs, examples, swc etc...
(This folder is not available in the zip file.)

Dependencies

This module is dependent on the core, common, utils, data, UI, UI Extended and TweenLite modules. The SWCs of the these modules can be found in the /lib folder.

Getting started

Adding the CodeComponents module to your project

To add the CodeComponents module to you project you could use the SWC file or the AS files. The SWC file can be found in the /bin folder. If you want to use the SWC file you need to add the SWC file to the class path of your project. You also need to add the dependent SWC's to the class path of your project. These SWCs

If you want to use the AS files you just need to copy the content of the /source folder to the source folder of your project. Or you could add the /source folder to the class patch of your project.
NOTE: Don't copy folders from an SVN check out, since this can cause SVN problems. It would be better to do an SVN export to your own folder.
