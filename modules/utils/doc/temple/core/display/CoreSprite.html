Classpublic class CoreSprite
InheritanceCoreSprite flash.display.Sprite
Implements ICoreDisplayObjectContainer, ISprite
Subclasses PerformanceStat

Base class for all Sprites in the Temple. The CoreSprite handles some core features of the Temple:
  • Registration to the Registry class.
  • Global reference to the stage trough the StageProvider.
  • Corrects a timeline bug in Flash (see http://www.tyz.nl/2009/06/23/weird-parent-thing-bug-in-flash/).
  • Event dispatch optimization.
  • Easy remove of all EventListeners.
  • Wrapper for Log class for easy logging.
  • Completely destructible.
  • Automatic removes and destruct children, grandchildren etc. on destruction.
  • Tracked in Memory (of this feature is enabled).
  • Some useful extra properties like autoAlpha, position and scale.

You should always use and/or extend the CoreSprite instead of Sprite if you want to make use of the Temple features.

See also

temple.core.Temple.registerObjectsInMemory

destruct()method 
public function destruct():void

Destructs the object and makes it available for garbage collection.

When overriding this method, always call super.destruct() at the end!

If you want the object to be available for garbage collection make sure you:
  • Remove all event listeners on this object (use removeAllEventListeners on Temple objects).
  • Remove all event listeners from this object.
  • Set all non-primitive variables to null.
  • Set all references to this object to null in other objects.
When a Temple object is destructed a DestructEvent.DESTRUCT is dispatched from the object (if the object implements ICoreEventDispatcher).

Note: Bear in mind that it is possible an object can be destructed more than once.
