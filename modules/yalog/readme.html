Yalog Module
 

About

This is version 3.0.0 of the Temple Yalog Module, built on 2012-02-17. Get the latest version from: http://code.google.com/p/templelibrary/. You can get the files from SVN or you could download it as a zip from the downloads page.

Yalog (Yet Another Logging application for Actionscript development) is a logging tool that comprises a set of classes for inclusion in an Actionscript application, and a viewing application for viewing the generated log messages. This can be very handy for debug purposes. Messages send with Yalog can be viewed with Yala and Yalala (which is an extended version of Yala). We recommend Yalala since Yalala is adjusted to the Temple version of Yalog. Yalala also has some more features like filtering and exporting messages.

Yalog is originally build by Stephan Bezoen (Acid Cats). The Temple version of Yalog is adapted by MediaMonks for better integration with the Temple and has some more features, like data compressing and named channels.

You can send messages directly to Yalog, or you could use the YaLogConnector which automatically send all Temple Log messages to Yalog. Check out the examples for more information about the usage of Yalog.

Content

This module contains the following folders:

/bin
Contains the SWC of this module.
/doc
Contains the documentation of the Yalog module.
/examples
Contains some examples of the Yalog module.
/lib
Contains the SWC files of other Temple modules which are needed for this module.
/source
Contains the source (.as) files of this module.
/tools
Contains ANT scripts which are used to generate docs, examples, swc etc...
(This folder is not available in the zip file.)

Dependencies

This module is dependent on the core and the common modules. The SWCs of the these modules can be found in the /lib folder.

Getting started

Adding the Yalog module to your project

To add the Yalog module to you project you could use the SWC file or the AS files. The SWC file can be found in the /bin folder. If you want to use the SWC file you need to add the SWC file to the class path of your project. You also need to add the dependent SWC's to the class path of your project. These SWCs can be found in the /lib folder.

If you want to use the AS files you just need to copy the content of the /source folder to the source folder of your project. Or you could add the /source folder to the class patch of your project.
NOTE: Don't copy folders from an SVN check out, since this can cause SVN problems. It would be better to do an SVN export to your own folder.

Examples

Check out the /examples folder for all examples. This folder also contains a list of all available examples.
