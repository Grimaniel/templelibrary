InheritanceDragBehavior BoundsBehavior AbstractDisplayObjectBehavior AbstractBehavior CoreEventDispatcher flash.events.EventDispatcher
Implements IEnableable

The DragBehavior makes a DisplayObject draggable. The DragBehavior uses the decorator pattern, so you won't have to change the code of the DisplayObject.

It is not nessessary to store a reference to the DragBehavior since the DragBehavior is automatically destructed when the DisplayObject is destructed.

destruct()method
override public function destruct():void

Destructs the object and makes it available for garbage collection.

When overriding this method, always call super.destruct() at the end!

If you want the object to be available for garbage collection make sure you:
  • Remove all event listeners on this object (use removeAllEventListeners on Temple objects).
  • Remove all event listeners from this object.
  • Set all non-primitive variables to null.
  • Set all references to this object to null in other objects.
When a Temple object is destructed a DestructEvent.DESTRUCT is dispatched from the object (if the object implements ICoreEventDispatcher).

Note: Bear in mind that it is possible an object can be destructed more than once.

Examples

If you have a MovieClip called 'mcClip' add DragBehavior like:

     new DragBehavior(mcClip);

If you want to limit the dragging to a specific bounds, you can add a Rectangle. By adding the Reactangle you won't be able to drag the DisplayObject outside the Rectangle:

     new DragBehavior(mcClip, new Rectangle(100, 100, 200, 200);

It's also possible to define a child object as a 'DragButton'. A DragButton is a DisplayObject that is used to drag the target.

If you have a MovieClip called 'mcClip' with a child called 'mcDragButton' add DragBehavior like:

     new DragBehavior(mcClip, null, mcClip.mcDragButton);
