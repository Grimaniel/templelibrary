Classpublic class MultiStateElement
InheritanceMultiStateElement CoreMovieClip flash.display.MovieClip
Implements IDebuggable

A MultiStateElement can be used inside a MultiStateButton. Is will automatically respond to the state of the MultiStateButton and will show the same state.

A MultiStateElement can be nested inside objects which has a ButtonBehavior. Nesting means that it is placed on the timeline of an object. But also on the timeline of a child or grand-child of the object. There can be an unlimited amount of sub-children between the object and the MultiStateElement.

The Temple knows different kinds of buttons. Check out the button schema in the UI Module of the Temple for a list of all available buttons which their features.

destruct()method
override public function destruct():void

Destructs the object and makes it available for garbage collection.

When overriding this method, always call super.destruct() at the end!

If you want the object to be available for garbage collection make sure you:
  • Remove all event listeners on this object (use removeAllEventListeners on Temple objects).
  • Remove all event listeners from this object.
  • Set all non-primitive variables to null.
  • Set all references to this object to null in other objects.
When a Temple object is destructed a DestructEvent.DESTRUCT is dispatched from the object (if the object implements ICoreEventDispatcher).

Note: Bear in mind that it is possible an object can be destructed more than once.
